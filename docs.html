GEO Workbench Lab Docs

This documentation describes the practical workflow and evidence structure used in the GEO Workbench Lab pilot.

1. Workflow steps

  1. Create a public target page.
  2. Validate URL, robots.txt, sitemap, canonical, and noindex state.
  3. Run before crawl/extraction/audit.
  4. Build a prompt pack.
  5. Collect baseline AI answers.
  6. Parse answers into a ledger.
  7. Generate prompt-to-page gap map.
  8. Generate claim-safe patch package.
  9. Apply and publish after human approval.
  10. Run after crawl/extraction/audit.
  11. Retest the same prompts.
  12. Compare page-quality and AI-answer metrics separately.
  13. Diagnose discoverability/entity gaps if AI lift is not observed.

3. Prompt pack format

A prompt pack should include:

  • prompt ID;
  • prompt text;
  • intent category;
  • priority;
  • platform;
  • expected evidence;
  • notes.

Prompt categories should include discovery, recommendation, comparison, pricing/cost, trust, risk/limitations, alternatives, use case, buying decision, and citation-seeking prompts.

4. Answer ledger schema

Each answer row should include:

  • test_id;
  • run_id;
  • timestamp;
  • platform;
  • model_name;
  • prompt_id;
  • exact_prompt;
  • answer_text_path;
  • screenshot_path;
  • mentioned_brand;
  • recommended_brand;
  • cited_brand_url;
  • cited_urls;
  • target_url_accuracy;
  • competitors_mentioned;
  • competitor_above_us;
  • rank_position;
  • recommendation_strength_0_5;
  • sentiment;
  • reason_not_recommended;
  • missing_evidence_hints;
  • source_type;
  • evidence_quality_score;
  • result_status;
  • blocker_reason;
  • notes.

7. Claim safety rules

Do not claim:

  • ranking guarantees;
  • citation guarantees;
  • ChatGPT recommendation lift unless measured;
  • customer adoption without evidence;
  • unsupported accolades;
  • formal credential claims;
  • formal security or compliance status;
  • production deployment status or enterprise deployment readiness status without evidence.

Allowed claims include self-owned pilot status, workflow description, page-quality improvement, and no-lift observation.

8. Manual answer capture rules

When collecting AI answers manually:

  1. Use the exact prompt text.
  2. Prefer one new chat per prompt.
  3. Do not tell the model the target URL unless the test is a direct-access diagnostic.
  4. Save full raw answer text.
  5. Save screenshot.
  6. Record platform, model, timestamp, and source URLs.
  7. Keep failed or blocked rows in the ledger.

9. Retest schedule

Recommended retest schedule after supporting assets are published:

  • direct URL diagnostic immediately;
  • search index checks immediately and again after indexing delay;
  • original 12 prompts after 3 days;
  • original 12 prompts after 7 days;
  • original 12 prompts after 14 days.
